Netlify

Jag har valt att använda Netlify som publicerings-plattform till denna uppgiften.

Netlify är en plattform som tillhandahåller webbhotell.

Netlify förenklar processen för utvecklare att distrubera och hantera för en webbplats, vilket innebär att man inte behöver lägga ner lika mycket tid på exempelvis överföring av filer och färdigställane av webbplats.

Netlify fungerar genom GitHub, där Netlify skapar sitt eget typ av arkiv genom ens olika repon. Därefter levereras förbyggda, statiska webbplatser till besökare genom CDN.

De flesta funktionerna i Netlify är gratis men det går att uppdatera till "Pro" och "Enterprise". De paketen har lite fler funktioner så som bakgrund, lösenordsskydd och E-postsupport. Paketen kostar från 19$ per månad.

Netlify stöder de flesta programmeringsspråken och ramverken, bland annat:

  • JavaScript
  • HTML
  • CSS
  • Node.js
  • Python
  • React
  • Vue.js

Fördelar

  • Netlify är ett billigt och snabbt alternativ till en färdig webbplats.
  • Man kommer långt genom att använda gratisfunktionerna.
  • Med hjälp av Git och de olika grenarna(branches) i Git är det enkelt att hålla koll och uppdatera sin webbplats

Nackdelar

  • Netlify är främst inriktad på statiska, frontendapplikationer och har därmed inga direkta backendfunktioner så som server eller databasverktyg.

Arbetssätt

Eftersom det var första gången jag använde Netlify så började jag med att skapa ett konto, som var kopplat till mitt GitHub.

När jag var färdig med att utveckla webbplatsen i Visual Studio Code och hade commitat samtliga filer till mitt GitHub så hämtade jag repot för webbplatsen via GitHub till en färdig webbplats i Netlify.

Fullstack-utveckling med ramverk.
